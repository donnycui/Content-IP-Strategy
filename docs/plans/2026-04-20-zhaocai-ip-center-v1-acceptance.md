# zhaocai-IP-center v1 Acceptance

## 产品定位

`zhaocai-IP-center v1` 是一个面向创作者本人的主流程中枢，不再是旧版研究工作台的集合。

这一版只保留两类内容：

1. 创作者主流程
   - IP 提炼
   - 创作者画像
   - 方向与选题
   - 风格与内容
   - 每日复盘
   - 升级进化
2. 系统设置
   - gateway 接入
   - 模型路由
   - 套餐权限

旧版 `signals / sources / reviews / research / drafts` 不再作为独立主产品页面对外暴露。它们如果仍有底层用途，也必须被隐藏在新工作流之后。

## 导航验收

顶部导航应只保留：

- 首页
- IP 提炼
- 创作者画像
- 方向与选题
- 风格与内容
- 每日复盘
- 升级进化
- 系统设置

不应继续出现以下一级导航：

- 信号研究
- 模型路由
- 套餐权限
- 演示路径
- 任何“工作台”字样

## 首页验收

首页应只包含 5 个区块：

1. 当前判断
   - 只说明“你现在在哪一步”
   - 只给一个主动作

2. 六步主流程
   - IP 提炼
   - 创作者画像
   - 方向与选题
   - 风格与内容
   - 每日复盘
   - 升级进化

3. 常用动作
   - 只保留最常用的 3 个动作

4. 系统快照
   - 只显示最重要的长期信息

5. 系统观察
   - 只显示结论型热点和趋势摘要

首页不应再出现：

- 中枢协调器长段落
- 旧版工作台解释
- 大量卡片堆叠
- 原始搜索结果列表

## Agent 页验收

所有 Agent 页应采用统一结构：

1. 阶段标题
2. 当前判断
3. 这一阶段要产出什么
4. 正式工作区
5. 完成后去哪

不应再出现：

- 旧页面 / legacy tools / v2.0 工作流文案
- 线程底座说明
- 仅用于解释系统结构的冗余卡片

## 各 Agent 功能验收

### 1. IP 提炼

页面目标：
- 让用户通过多轮对话完成 IP 提炼

必须存在：
- `Brainstorming` 开关：`OFF / AUTO / ON`
- 对话式提炼入口
- 当前对话记录
- 用户回答输入框
- 生成画像草案动作

必须不存在：
- 快捷提炼模式切换
- 旧提炼工作台壳层

### 2. 创作者画像

页面目标：
- 固化和编辑创作者长期画像

必须存在：
- 画像编辑表单
- 当前阶段、活跃方向、主题数量、待确认建议等摘要指标

### 3. 方向与选题

页面目标：
- 在一个工作区里完成方向、主题线和今日选题

必须存在：
- 研究与趋势区块
- 方向/主题/选题的统一工作区
- 方向生成、主题生成、选题生成动作

不应再拆成多个独立旧页面作为主入口。

### 4. 风格与内容

页面目标：
- 学习风格并产出内容资产

必须存在：
- style skill 摘要
- 样本上传
- style revision
- 内容项目列表或内容工作区

### 5. 每日复盘

页面目标：
- 录入内容表现并形成复盘结论

必须存在：
- 复盘数据录入
- 复盘摘要

### 6. 升级进化

页面目标：
- 消费复盘和学习结论，形成更新建议

必须存在：
- 主动学习结论
- 进化建议
- 平台策略建议

## 搜索 / 热点 / 学习能力验收

这部分不应是独立一级页面。

正确放置位置：

- 首页：只显示摘要
- 方向与选题：作为研究与趋势区块
- 升级进化：作为长期学习结论输入

## 旧页面重定向验收

以下页面访问时，应进入新的 Agent 路由，而不是停留在旧页面壳层：

- `/profile/extract`
- `/profile`
- `/directions`
- `/topics`
- `/candidates`
- `/reviews`
- `/signals`
- `/sources`
- `/evolution`
- `/research/[id]`
- `/drafts/[id]`

## 系统设置验收

系统设置中应保留：

- Gateway Access
- Alias Catalog
- Capability Routing
- Plan Access

当前线上模型主链应已切到 `zhaocai-gateway`，不再由 `openai-primary` 承担主能力路由。
