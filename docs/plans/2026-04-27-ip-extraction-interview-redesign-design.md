# IP 提炼并行发散访谈设计

## 背景

当前 `IP提炼` 存在 3 个产品问题：

1. 开场太轻，只问双方称呼，无法快速建立有效上下文。
2. 访谈过早收尾，容易在一两轮后就进入“信息已经足够”。
3. 访谈维度不完整，后续内容生产真正需要的表达形式和平台风格偏好没有被系统摸清。

这次 redesign 的目标不是微调问法，而是把 `IP提炼` 从“轻量定位访谈”改成“并行发散式完整提炼”。

## 目标

新的 `IP提炼` 要覆盖这 5 类核心维度：

1. 方向
2. 人设
3. 受众
4. 表达形式
5. 平台风格偏好

系统默认采用 `强约束`，不把这 5 类维度全部触达到“已足够”，就不允许进入收尾态。

## 开场设计

第一轮不再只问双方怎么称呼，而是先拿到一段“带结构的自我介绍”。

开场引导用户一次性讲 4 件事：

1. 你现在是做什么的，职业背景是什么
2. 你想打造什么方向的 IP
3. 你现在对这个 IP 最初的想法是什么
4. 你为什么现在想做这件事

双方称呼仍然保留，但从“唯一开场内容”降级成“开场的一部分”。

## 访谈策略

新的访谈不是线性表单，而是“并行发散 + 有优先级”。

### 优先级顺序

1. 方向与价值
2. 人设与可信度
3. 受众与使用场景
4. 表达形式
5. 平台风格偏好

### 规则

系统每次仍然只问一个最有价值的问题，但内部同时维护这 5 个维度的状态。

每轮回答后，系统要先判断：

1. 这轮更新了哪些维度
2. 哪个维度最关键、最缺、最影响后续内容生产
3. 下一问应该继续深挖、补空白，还是交叉发散

### 并行发散的具体含义

并行发散不等于乱问。它的含义是：

- 不要求“方向讲完再讲形式”
- 用户如果一开始就明显带着形式偏好或平台偏好，系统可以提前切入
- 只要它能帮助尽快摸清完整画像，就允许在不同维度之间跳转

## 约束档位

新增一个前端可见控制项：

`提炼约束`

可选值：

- `强约束`
- `中等约束`
- `弱约束`

默认：

- `强约束`

说明文案：

`默认强约束，系统会尽量把方向、人设、受众、表达形式和平台风格偏好都聊到。`

### 使用方式

- 开始前可设置
- 对话过程中也允许切换

## 结束判定

系统内部对每个维度维护 3 种状态：

1. `未触达`
2. `已触达但不够`
3. `已足够`

### 强约束

只有 5 个维度全部达到 `已足够`，才允许进入收尾。

### 中等约束

满足下面两条才允许收尾：

1. `方向 / 人设 / 受众` 全部 `已足够`
2. `表达形式 / 平台风格偏好` 至少 1 个 `已足够`，另 1 个至少 `已触达`

### 弱约束

满足下面两条即可收尾：

1. `方向 / 人设 / 受众` 至少 2 个 `已足够`
2. `表达形式 / 平台风格偏好` 至少 1 个 `已触达`

### 最小轮数

为了避免“一条介绍就结束”，还要加最小有效轮数限制：

- `强约束`：至少 `6-8` 轮
- `中等约束`：至少 `4-6` 轮
- `弱约束`：至少 `3` 轮

## 关键实现约束

### 1. 不允许只因为模型说够了就结束

模型的 `readyToFinalize` 只能作为参考，不能直接决定收尾。

最终是否允许结束，必须经过本地结束判定器。

### 2. 不允许只因为出现“如果你愿意”这类措辞就强制收尾

当前把这类措辞直接改写成收尾提示的逻辑过于激进，需要降级。

### 3. 草稿字段必须容忍非字符串返回

模型可能返回：

- 字符串
- 数组
- 对象

归一化层必须先做安全转换，不能再出现 `trim is not a function` 这类错误。

## 预期结果

这次改完后，`IP提炼` 应该体现出下面这些变化：

1. 开场更完整，不再只围绕称呼打转
2. 中间访谈更充分，不会过早结束
3. 输出更接近后续内容生产真实需要的输入
4. 约束强度可控，既能深聊，也能按需放松
5. 用户即使补充新信息，也不会因为字段类型问题把整轮对话炸掉
