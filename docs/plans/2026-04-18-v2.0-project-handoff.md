# Content IP Research Workbench v2.0 Handoff

**Date:** 2026-04-18  
**Version:** `v2.0.0`  
**Project:** `content-ip-research-workbench`

## 1. Version Definition

This project should now be treated as **v2.0**.

The meaning of `v2.0` in this repo is:

- Provider and gateway routing have moved beyond the original prototype stage
- Admin configuration is no longer only a scaffold
- Homepage and key working pages have started performance-oriented restructuring
- Conversational profile extraction has been introduced as the next major workflow direction

This is still an actively evolving product, but it is no longer a raw `0.1.x` prototype.

## 2. Current Product State

The project currently includes these major capability areas:

- Creator profile extraction and editing
- Directions / topics / candidates workflow
- Signal ingestion, scoring, review, and research progression
- Draft generation and editing
- Admin management for gateway access, models, capability routing, and plans

The most recent structural work has focused on:

- Gateway alias cutover support
- Provider/admin management hardening
- Performance improvements on homepage and major workbench pages
- Conversational profile extraction foundation

## 3. Major Landed Changes

### 3.1 Gateway alias routing

The project has already introduced gateway-alias-oriented routing support and the corresponding handoff docs.

Relevant historical docs:

- `docs/plans/2026-04-02-gateway-alias-cutover-handoff.md`
- `docs/plans/2026-03-31-gateway-alias-cutover.md`

### 3.2 Admin hardening

Provider / model management has been extended to support:

- safe enable / disable
- safe delete checks
- manual model creation
- route-usage-aware UI restrictions

### 3.3 Performance restructuring

Performance work has already started for key pages:

- homepage split into independently loading sections
- signals page split into forms vs. table sections
- topics and candidates pages split into profile vs. list sections
- admin pages moved away from blanket `force-dynamic` toward short revalidation

### 3.4 Conversational profile extraction

A first implementation of conversational profile extraction is now in the repo:

- session persistence in Prisma
- conversation APIs
- dual-mode extraction UI

It is not fully product-polished yet, but it is now a real code path rather than just a concept.

## 4. Current Risks / Incomplete Areas

These are the most important unfinished or unstable areas:

1. Conversational profile extraction still needs real-world UX iteration
2. Some page-level performance issues remain beyond the first optimization pass
3. Gateway vs. direct-provider strategy is still a live architectural discussion
4. Manual smoke tests are still required after major workflow changes

## 5. Recommended Operator Priorities

When resuming work from this handoff, prioritize in this order:

1. Validate the active workflow you are changing
2. Keep project routing / gateway configuration coherent
3. Prefer incremental page-speed improvements over broad rewrites
4. Keep documenting active decisions in `docs/plans/`

## 6. Active Handoff Rule

From this point forward, treat **this document** as the active high-level handoff for the repo unless a newer dated `v2.x` handoff replaces it.
